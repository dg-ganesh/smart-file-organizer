Apache parquet: (developed by twitter and cloudera)
USer columnar storage, (whereas excel, JSON etc store info on row based format)
IT uses WORM format, write only read many time format.

The normal data format is stored in row, e.g. if the data is Name; age
e.g
Ganesh	40
Arnav	10
Raksha	30
The CSV stores in its db as 
Ganesh 40, arnav 10, raksha 30 etc. 

Instead if parquet then it stores as 
Ganesh Arnav Raksha; 40 10 30.
This helps in quicker retrival of stored data.

Redshit spectrum:

BAtch vs streaming
Data can be ingested in mode of batch or it can be ingested as stream. sreaming is used mainly in cases where the live data related action is very important. e.g. map informaiton. 

Kinesis: is the tool provided by AWS for streaming
Service in Kinesis:

Kinesis firehose: collects and stores data and can be used for S3, redshit etc. This scales automatically

Athena:
========
This can direclty query data on the S3 or any data file. This is only charged on the basis on query run. REcommended for Adhoc data discovery

IN the athena main window, select the DB, it will list the catellogue information as pulled in by glue, 
You can run default predefine queyr (like list first 10 row etc) 
it shows window similar to SQL query. 

This basically means you dont need to provision serves to query the data which is stored blindly in S3. 

Athena basically backs on the catelog built by Glue, so its important to ensre glue is regularly run

CSV works great if the col structure is not changed meaning if no new col are added or if name change etc. Else Athna will fail. 

But in parquet format the col numbe is not stored but it is the name of the col that is stored. This means if the col changes it will not impact.
but again if the col name are changed then athena will return blank.

PArtitioning:
is an important technique to ensure the queries are run quickly and cost effectively. In this case there are sepreate folders crated for each partitioning and when we can the queries can either collate altogether or query based on partitinins. but the cateloges needs toe be updated regularly when the partitions are updated 

The catelog can also be maintained manually. 

IF the parquet are to be maintained in the folder based on hive folder structure.
Then the name of the partition folder will be in key; value pairs. instead of just names
You can further design more sub folder to perform further partitions.
